vue项目初级优化过程

aliken 2019-05-12 vue

项目已经开发到一定阶段了,需求基于稳定，之前业务方一直抱怨，为什么页面加载这么慢，现在终于有时间来优化下项目了。通过博客来记录这一次的过程。

# 一、从代码上排查加载慢的原因

首先想到的是首页加载的文件是不是首页必须，首页加载的文件大小和数量是不是合理的。通过 chrome network（fast 3G 来大体模拟线上环境的网络）工具，查看加载的文件(js文件)。截图如下： 通过分析我们了解到，文件数量4个、大小568k、总加载时间为5.23s(比较严重)。 我首先想到的是能否减小文件体积，通过 webpack-bundle-analyzer 插件来排查首页是否加载了不需要的代码。 上面截图是整体的包引入情况 我们依次分析每个文件包的情况，是否能优化：

  • app 包基本上都是必须代码没什么可优化的
  • vendors iview 目前看着是全部引入，可以改为按需引入。引入babel-plugin-import插件，创建代码，写入如下配置（参考iview 官方文档）



 




{
  {
  "plugins": [["import", {
    "libraryName": "view-design",
    "libraryDirectory": "src/components"
  }]]
}

添加如上配置后vendors 打包如下 通过查看，iview已经明显减小（1.6M -> 580K)将近三分之二，按需引用可以很大程度上剔除掉项目不需要的代码。继续我们的优化通过截图我们可以发现现在vendors包里面比较大的是vxe-table(一个处理大数据table组件)。首先试试能否通过按需加载的方式，减小vxe-table,通过组件提供的文档我们发现此组件支持按需加载。 babel 配置改为如下babel-plugin-import 官方说明




 










{
  {
  "plugins": [["import", {
    "libraryName": "view-design",
    "libraryDirectory": "src/components"
  },'view-design'],
  [
    "import",{
      "libraryName":"vxe-table",
      "style":true
    },"vxe-table"]
  ]
}

修改完如上配置后，vendors 打包如下 vxe-table 组件大小从800k减为300K，到现在为止iview+vxe-table 两个组件总共减少1.4M。通过继续分析，vendors 里面剩余的包基本上都是vue 框架和代码监控代码，几乎是不能再瘦身了，到此为止，vendors瘦身结束。

  • 剩下的chunk 文件，基本上没什么可瘦身的 启动一下看看通过瘦身，性能提升了多少，截图如下 怎么才快了1.4S???? 继续搞.... 因为webpack打包的时候默认压缩了代码，所以下一步搞CDN（cdn 一般自带GZIP服务 又能进一步压缩代码 爽爽爽...）上传代码到公司CDN服务器（项目代码一定要上到企业CDN服务，不要上传第三方的CDN，个人代码除外）CDN 上传服务 一般都会有上传配置/脚本，这里就不贴CDN上传脚本了。
